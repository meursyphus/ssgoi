# 뎁스(Depth) 전환

뎁스(Depth) 전환은 **Z축 계층 이동**을 표현하는 전환입니다. Material Design의 Shared Z-Axis transition과 동일한 개념으로, 두 화면이 같은 방향으로 scale과 fade를 조합하여 "깊이감"을 만들어냅니다.

## UX 원칙

### 언제 사용하나요?

뎁스 전환은 **앱의 계층 구조에서 한 단계 위 또는 아래로 이동**할 때 사용합니다. 작은 트리거(아이콘, 버튼)에서 전체 화면으로 확장되는 경험을 자연스럽게 만들어줍니다.

#### 주요 사용 케이스
- 검색 아이콘 → 검색 화면 (작은 아이콘에서 전체 화면으로)
- 설정 아이콘 → 설정 화면 (트리거에서 상세 화면으로)
- 알림 아이콘 → 알림 목록 (요약에서 상세로)
- 카테고리 → 하위 카테고리 (계층적 네비게이션)
- 상품 목록 → 상품 상세 (목록에서 깊이 들어가기)

### 왜 이렇게 동작하나요?

1. **전진 이동 (enter)**: 새 화면이 앞쪽에서 작게 시작하여 확대되며 나타나고, 이전 화면은 뒤쪽으로 밀려나며 확대되어 사라집니다. 이는 "깊이 들어가는" 느낌을 줍니다.

2. **후진 이동 (exit)**: 현재 화면이 앞쪽으로 축소되며 사라지고, 이전 화면은 뒤에서 축소되어 나타납니다. 이는 "깊이에서 빠져나오는" 느낌을 줍니다.

3. **계층적 이해**: Scale의 방향이 일관되게 움직여 사용자가 현재 어느 계층에 있는지 직관적으로 파악할 수 있습니다.

### 모션 디자인

**전진 (enter) 동작**:
- **나가는 페이지**: scale 1 → 1.2 + fade out (뒤로 확대되며 사라짐)
- **들어오는 페이지**: scale 0.8 → 1 + fade in (앞으로 나오며 나타남)

**후진 (exit) 동작**:
- **나가는 페이지**: scale 1 → 0.8 + fade out (앞으로 축소되며 사라짐)
- **들어오는 페이지**: scale 1.2 → 1 + fade in (뒤에서 앞으로 나타남)

이 모션은 **물리적 깊이감**을 시뮬레이션하여 사용자에게 공간적 맥락을 제공합니다.

## 기본 사용법

### 1. 전환 설정

::code-group

```tsx [React]
import { Ssgoi } from '@ssgoi/react';
import { depth } from '@ssgoi/react/view-transitions';

<Ssgoi
  config={{
    transitions: [
      {
        from: '/search-icon',
        to: '/search',
        transition: depth({ direction: 'enter' }),
        symmetric: true // 뒤로가기 시 자동으로 exit 방향 적용
      },
      {
        from: '/settings-icon',
        to: '/settings',
        transition: depth({ direction: 'enter' }),
        symmetric: true
      }
    ]
  }}
>
  {children}
</Ssgoi>
```

### 2. 방향 설정

`direction` 옵션으로 계층 이동 방향을 지정합니다:

- **`enter`**: 깊이 들어가기 (기본값)
  - 새 화면이 앞에서 확대, 이전 화면은 뒤로 확대
  - 하위 레벨로 이동할 때 사용
- **`exit`**: 깊이에서 나오기
  - 현재 화면이 앞으로 축소, 이전 화면은 뒤에서 축소
  - 상위 레벨로 돌아갈 때 사용

```typescript
// 하위 레벨로 들어갈 때
depth({ direction: 'enter' })

// 상위 레벨로 돌아갈 때
depth({ direction: 'exit' })

// symmetric: true를 사용하면 자동으로 반대 방향 생성
```

## 커스터마이징

### 스프링 물리 설정

뎁스 전환의 스프링 애니메이션을 커스터마이징할 수 있습니다:

```typescript
depth({
  direction: 'enter',
  spring: {
    stiffness: 300,  // 스프링 강성 (높을수록 빠름)
    damping: 25,     // 감쇠 (높을수록 덜 튐)
    mass: 1          // 질량 (높을수록 느림)
  }
})
```

### Scale 범위 조정

Scale의 시작/종료 값을 조정하여 깊이감의 강도를 변경할 수 있습니다:

```typescript
// 더 강한 깊이감
depth({
  direction: 'enter',
  scaleRange: { min: 0.7, max: 1.3 }
})

// 더 부드러운 깊이감
depth({
  direction: 'enter',
  scaleRange: { min: 0.9, max: 1.1 }
})
```

## 주의사항

### symmetric 옵션 활용

뎁스 전환은 양방향으로 동작하므로 `symmetric: true`를 사용하면 편리합니다:

```typescript
{
  from: '/list',
  to: '/detail',
  transition: depth({ direction: 'enter' }),
  symmetric: true
  // 자동으로 /detail → /list는 exit 방향으로 동작
}
```

### 계층 구조 명확히

뎁스 전환은 **명확한 계층 관계**가 있을 때만 사용하세요. 동일 레벨의 화면 간 이동에는 슬라이드나 페이드가 더 적합합니다.

### 성능 고려

Scale과 opacity 애니메이션은 GPU 가속을 받지만, 복잡한 레이아웃에서는 성능 테스트가 필요합니다.

사용자의 모션 감소 설정을 존중하세요:

```css
  /* SSGOI는 자동으로 전환 시간을 단축합니다 */
}
```

### 포커스 관리

페이지 전환 후 적절한 요소로 포커스를 이동시키세요:

```typescript
// 새 페이지 로드 후
useEffect(() => {
  // 메인 콘텐츠로 포커스 이동
  document.querySelector('h1')?.focus();
}, []);
```

## 모범 사례

### ✅ 좋은 예

- 검색 아이콘 → 검색 전체 화면
- 상품 카드 → 상품 상세 페이지
- 설정 버튼 → 설정 화면
- 알림 벨 → 알림 목록
- 카테고리 → 하위 카테고리

### ❌ 나쁜 예

- 탭 간 이동 (형제 관계) - 슬라이드 사용
- 독립적인 섹션 간 이동 - 페이드 사용
- 모달 열기 - 시트 전환 사용
- 뒤로가기 없는 일회성 화면 - 드릴 사용
