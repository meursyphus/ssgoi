# 커튼 전환

커튼 전환은 블라인드나 커튼이 닫히고 열리는 듯한 시각적 효과로 화면을 전환합니다. 여러 개의 막대가 순차적으로 화면을 가리고 드러내며, 무대의 막이 오르내리는 듯한 극적인 연출을 제공합니다.

## UX 원칙

### 언제 사용하나요?

커튼 전환은 **극적인 전환(Dramatic Transition)** 이나 **중요한 순간 강조** 시 사용됩니다.

#### 주요 사용 케이스

- **온보딩 플로우**: 단계별 튜토리얼이나 소개 화면
- **결과 공개**: 테스트 결과, 점수 발표 등 긴장감 있는 순간
- **스토리텔링**: 내러티브가 있는 콘텐츠 전환
- **프레젠테이션 모드**: 슬라이드쇼나 갤러리 모드

### 왜 이렇게 동작하나요?

1. **시선 집중**: 순차적인 블라인드 애니메이션이 사용자의 주의를 끌어당깁니다
2. **기대감 조성**: 콘텐츠가 점진적으로 드러나며 궁금증을 유발합니다
3. **명확한 전환**: 화면이 완전히 가려졌다가 열리며 확실한 구분을 제공합니다

### 모션 디자인

```
나가는 화면: 블라인드가 왼쪽에서 오른쪽으로 순차적으로 닫힘
짧은 대기: 완전히 가려진 상태 유지
들어오는 화면: 블라인드가 오른쪽에서 왼쪽으로 순차적으로 열림
```

이 시퀀스는 무대의 막이 닫히고 열리는 듯한 극장적 경험을 제공합니다.

## API 레퍼런스

### CurtainOptions

```typescript
interface CurtainOptions {
  // 스프링 설정
  inSpring?: SpringConfig; // IN 애니메이션 스프링 (기본: { stiffness: 300, damping: 22 })
  outSpring?: SpringConfig; // OUT 애니메이션 스프링 (기본: { stiffness: 400, damping: 25 })

  // 타이밍 설정
  transitionDelay?: number; // OUT과 IN 사이 대기 시간 (기본: 50ms)
  staggerDelay?: number; // 블라인드 간 지연 시간 (기본: 30ms)

  // 시각적 설정
  blindCount?: number; // 블라인드 개수 (기본: 10)
  direction?: "horizontal" | "vertical"; // 블라인드 방향 (기본: 'horizontal')
  curtainColor?: string; // 커튼 색상 (기본: '#000000')
}
```

## 접근성

- 애니메이션이 진행되는 동안 포커스 이동이 자연스럽게 처리됩니다
- `prefers-reduced-motion` 설정 시 애니메이션이 단순화됩니다
- 스크린 리더는 전환을 무시하고 콘텐츠 변경만 인식합니다

## 성능 고려사항

- 블라인드는 동적으로 생성되고 애니메이션 완료 후 자동 제거됩니다
- GPU 가속을 활용하여 부드러운 애니메이션을 제공합니다
- 많은 수의 블라인드(20개 이상)는 저사양 기기에서 성능 영향을 줄 수 있습니다

## 모범 사례

### ✅ DO

- 중요한 순간이나 특별한 콘텐츠 전환에 사용
- 브랜드 아이덴티티에 맞게 커스터마이징
- 적절한 블라인드 개수로 성능과 효과의 균형 유지

### ❌ DON'T

- 빈번한 일반 네비게이션에는 사용하지 마세요
- 너무 많은 블라인드로 과도한 효과 자제
- 빠른 연속 전환이 필요한 곳에는 부적합
